# Task 20: 残りの 1000 行超ファイルの分割

## フェーズ

Phase 4（中長期）- P0-4 対応・残件対応

## 背景

Phase 4 の目標は 1000 行超ファイル数を 17 → 8 以下にすること。Task 16〜19 で主要 4 領域を対応した後、まだ目標に達していない場合に実施する。

## 実施内容

1. 現時点で 1000 行超のファイルを列挙する：
   ```bash
   find src -name "*.cpp" -o -name "*.h" | xargs wc -l | sort -rn | head -20
   ```

2. 1000 行超のファイルのうち、未対応のものを特定する

3. 各ファイルについて、以下の方針で分割する：
   - ヘルパークラス抽出
   - 部分ファイル分割（同一クラスの `.cpp` を責務ごとに分割）
   - Presenter/Coordinator パターンでのロジック分離

4. 分割を実施し、ビルド確認する

## 完了条件

- 1000 行超ファイル数が 8 以下
- ビルドが通る

## 前提

- Task 16〜19 が完了していること

## 注意事項

- 全ファイルを 1000 行以下にする必要はない。目標は 8 以下に収めること
- `settingsservice.cpp` は Task 15 で対応済みのはず
- テストファイルは対象外
