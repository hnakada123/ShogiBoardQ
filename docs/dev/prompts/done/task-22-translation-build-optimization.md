# Task 22: translations ビルドターゲット最適化

## フェーズ

Phase 5（継続運用）- P2-2 対応

## 背景

`translations` ターゲットが `ALL` に含まれており、本体ターゲットが常に `translations` に依存する。通常開発ビルドの実行時間に翻訳更新コストが常時加算される。

## 実施内容

1. `CMakeLists.txt` の翻訳関連セクション（743行付近）を確認する

2. 以下のいずれかの方法で最適化する：
   - **案A**: `translations` を `ALL` から外し、明示的に `cmake --build build --target translations` で実行する方式
   - **案B**: CMake オプション `ENABLE_TRANSLATIONS`（デフォルト ON）で制御する方式
   - **案C**: 翻訳ファイルの変更検出を改善し、不要な再生成をスキップする方式

3. 選択した方式を実装する
4. ビルド確認:
   - `cmake --build build` で翻訳なしビルドが動作すること
   - 翻訳を含むビルドも引き続き可能であること
5. CLAUDE.md の Build Commands セクションを必要に応じて更新する

## 完了条件

- 通常の `cmake --build build` で翻訳生成がスキップされる（または高速化される）
- 翻訳を含むビルドが引き続き可能
- ビルドが通る

## 注意事項

- リリースビルドでは翻訳が確実に含まれるようにする
- CI パイプラインがある場合は影響を考慮する
