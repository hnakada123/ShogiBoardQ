# Task 19: デッドシグナル/スロット棚卸し・削除

未使用のシグナルとスロットを特定し、削除してください。

## 作業内容

### Step 1: シグナルの棚卸し

コードベース全体で `signals:` セクション内のシグナル宣言を全て列挙し、各シグナルについて:

1. `emit シグナル名` で発火箇所を検索
2. `connect(..., &Class::シグナル名, ...)` で接続箇所を検索
3. 以下の3カテゴリに分類:
   - **A. 正常**: emit あり + connect あり → 維持
   - **B. 未接続**: emit あり + connect なし → 将来使用の可能性を検討、不要なら削除
   - **C. 未発火**: emit なし + connect あり → バグの可能性、調査して修正または削除
   - **D. 完全未使用**: emit なし + connect なし → 削除

### Step 2: スロットの棚卸し

`public slots:` / `private slots:` セクション内のスロット宣言を全て列挙し、各スロットについて:

1. `connect(..., &Class::スロット名)` で接続箇所を検索
2. 直接呼び出し（`obj->スロット名()` 等）を検索
3. 以下に分類:
   - **A. 使用中**: connect あり、または直接呼び出しあり → 維持
   - **B. 未使用**: connect なし + 直接呼び出しなし → 削除

### Step 3: 削除の実行

カテゴリ D のシグナルとカテゴリ B のスロットを削除:
- ヘッダから宣言を削除
- `.cpp` から実装を削除
- 関連する `connect()` が残っていないことを確認

### Step 4: 結果の記録

削除したシグナル/スロットのリストを出力に含める。

## 既知のデッドシグナル候補

- `AnalysisCoordinator::analysisStarted` — emit されるが connect されていない（コメントで言及あり）

## 制約

- 判断に迷う場合は削除せず、コメントで `// TODO: verify usage` を付記
- 全既存テストが通ること
- UI の動作に影響がないこと
